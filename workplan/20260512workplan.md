## 背景說明

初始建構（2026-05-11）完成了所有頁面的 UI，但 API 層全部使用 Mock 資料。  
後端（.NET / ASP.NET Core）對應的端點已在 Vue 專案中實作並驗證。  
本次工作目標為：

1. **補齊 `hooks/` 資料夾**（技術選型要求但初始未建立）
2. **API 層切換至真實後端呼叫**（後端已實作的端點）
3. **型別定義對齊後端回傳結構**（`id` 從 `number` → `string`，角色/系統型別全面更新）
4. **頁面改用 hooks 存取資料**，移除直接引用 `MOCK_*` 的程式碼

後端**尚未實作**的端點維持 Mock：

- `getSessions()` / `revokeSession()` → `/sessions`
- `getAuditLog()` → `/audit`
- `restartContainer()` → `/system/containers/:name/restart`
- `DashboardPage` 活動流

---

## 架構分層（完成後）

```
pages/     ← 只管 UI 狀態（dialog, filter, query, 表單欄位）
  └─ hooks/  ← TanStack Query：useQuery / useMutation，管理 loading/error/data
       └─ api/  ← 純函式，只做 HTTP 呼叫，回傳 Promise，零副作用
```

## 技術決策

| 項目                    | 決定                                                                                                                                                |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| Hooks 命名              | `use{Domain}` 複數查詢，`use{Action}{Domain}` 寫入操作                                                                                              |
| TanStack Query key 規範 | `['users']`、`['roles']`、`['system', 'metrics']`、`['network', 'config']`                                                                          |
| 401 自動登出            | `client.ts` 回應攔截器：非 login 路徑的 401 → `authStore.logout()`                                                                                  |
| FormData Content-Type   | 上傳時刪除 Content-Type，讓瀏覽器自動設定 multipart boundary                                                                                        |
| SessionUser 欄位        | 補 `roles: string[]`、`permissions: string[]`、`preferredLocale: string \| null`                                                                    |
| Role 型別               | 對齊後端 `RoleDetailDto`：補 `displayName`、`isSystemRole`、`isActive`、`permissions[]`、`menuItemIds[]`                                            |
| SystemMetrics           | 使用後端 `/system/metrics` 統一端點，`Container` 從 `metrics.docker.containers` 讀取                                                                |
| NetworkAdapter          | 欄位名對齊後端：`interfaceName`、`displayName`、`macAddress`、`isPrimary`、`isConnected`、`connectionMode`、`ipAddress`、`subnetMask`、`dnsServers` |
| Mock 保留策略           | `systemApi.ts` 保留 sessions/audit/restartContainer 為 mock，等後端補齊                                                                             |

## 補丁紀錄

### BUG-01：開發環境登入 404

**發現時機：** Step 7 驗收後實際試跑時  
**根本原因：** `vite.config.ts` 缺少 `server.proxy` 設定。`/api/*` 請求打到 Vite dev server 本身而非後端，Vite 找不到路由 → 404。  
**對照：** Vue 專案（`web/vite.config.ts`）有設定 `proxy: { '/api': { target: 'http://localhost:5000' } }`，React 專案移植時漏掉。  
**修正：** 在 `web-react/vite.config.ts` 補上同樣的 proxy 設定，port 使用 `5173`。  
**影響範圍：** 僅開發環境；正式部署透過 nginx 轉發，不受影響。
